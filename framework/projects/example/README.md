# Example Project: Acme Customer Portal

This is an example project demonstrating how to use the SpecKit M365 Companion framework.
It shows the expected folder structure and document formats.

## Overview

The Acme Customer Portal is a fictional project used to illustrate the SpecKit workflow.
Review the files in this folder to understand:

- How to structure your project folders
- What documents look like after using `/constitution`, `/spec`, etc.
- Naming conventions for files

## What's Included

```
example/
├── README.md               ← This file (project overview)
├── sources/                ← Raw inputs (empty in this example)
├── docs/
│   └── 01_constitution.md  ← Sample constitution
└── notes/
    └── 2025-01-kickoff.md  ← Sample meeting notes
```

## How This Example Was Created

1. Copied `_template/` to `example/`
2. Renamed README with project overview
3. Used `/constitution` command to create the sample constitution
4. Added sample meeting notes to demonstrate the `notes/` folder

## Using This as a Reference

When starting your own project:

1. **Don't modify this folder** - keep it as a reference
2. **Copy `_template/`** to create your new project
3. **Compare your work** against this example to check format

## Next Steps for Your Project

After reviewing this example:

1. Copy `_template/` and rename it
2. Set up your Copilot Notebook with the start prompt
3. Run `/constitution` to create your own principles
4. Continue with `/spec`, `/plan`, `/tasks`
